OCREditor 官方操作手冊

本手冊依照 AI OCR Pro Editor 的實際畫面撰寫，說明檔案載入、原生 OCR、文字圖層編輯、框選 OCR、框選刪除、字型套用、AI 本地協同操作與匯出流程。

代表圖示與畫面一致：📄 互動圖層、🔄 重新辨識、ⓘ、➖、➕、100%、▸/▾。

1. 畫面區域與主要面板

📄左側：文件狀態與互動圖層

顯示圖片是否已載入、OCR 是否正在處理，以及所有可編輯文字圖層。點選任一圖層時，畫布會自動移動到該文字區塊中心。

▣中央：圖片畫布

畫布顯示原圖、OCR 文字框、背景遮罩與手動新增的文字。可縮放、拖曳檢視、點選文字框直接編輯。

⚙右側：格式、OCR 與常用操作

包含文字格式設定、主線原生 OCR、備援引擎、預設字型、AI 本地協同操作與常用操作按鈕。

2. 載入圖片與重新辨識

  • 📁從上方 檔案 選單選擇 載入圖片...，或在尚未載入圖片時點擊畫布中的 Open Image 按鈕。載入後會顯示 ✓ 圖片已載入並處理完成。
  • OCR預設主線是 主線引擎：原生 OCR。桌面瀏覽器可透過本地伺服器使用 Apple Vision / Windows OCR；iOS/Android App 可直接走手機端原生 OCR。
  • 🔄切換 OCR 設定或調整圖片後，按 🔄 重新辨識 會套用目前引擎重新產生文字圖層。
  • ▸右側 本地伺服器的用途 是可收合面板；點擊 ▸/▾ 可展開或收起，節省版面空間。

2.1 原生 OCR 無法連線：平台判斷與完整排除

先確認使用模式：Windows/macOS 桌面瀏覽器需要本機 OCR 伺服器；打包後的 iOS/Android App 直接呼叫裝置 OCR，不使用 localhost；手機瀏覽器無法啟動原生 OCR 伺服器，請改用 Gemini/Tesseract 或安裝打包 App。

平台正確連線方式
Windows Chrome / Edge執行 setup_and_run_ocr.bat，等出現已啟動提示後回網頁按「測試連接」；啟動完成後可關閉命令視窗。
macOS Chrome / Safari在專案資料夾執行 setup_and_run_ocr.sh，保持終端機開啟，再按「測試連接」。
iOS / Android App必須是 Capacitor 打包版本，直接使用 iOS Vision / Android ML Kit，不需要也不應填 localhost。
手機瀏覽器無法直接使用作業系統原生 OCR；iPadOS 的 Safari/Chrome 不能從網頁啟動 localhost，也不能直接呼叫 Apple Vision。請使用 Gemini/Tesseract，或安裝打包 App。

Windows 零基礎處理步驟

  1. 下載官方啟動檔：setup_and_run_ocr.bat。若公司網路封鎖單檔，改下載完整專案 ZIP。
  2. Chrome 警告時，只有在網址確認為 raw.githubusercontent.com/hauchiehlin-ops/ocr 後才選擇保留；SmartScreen 出現時選「其他資訊」→「仍要執行」。
  3. 開啟「命令提示字元」，解除下載檔案封鎖並執行：cd /d "%USERPROFILE%\Downloads"
    powershell -NoProfile -Command "Unblock-File -Path '.\setup_and_run_ocr.bat'"
    setup_and_run_ocr.bat
  4. 新版 BAT 會自動偵測並安裝 Python、建立隔離環境、下載 OCR 元件及設定登入後自動啟動；只有安全軟體或公司網路攔截時才需使用下列手動備援指令：winget install -e --id Python.Python.3.12
  5. 等 BAT 顯示 Windows Native OCR is ready 後，可關閉命令視窗；若要手動驗證，另開命令提示字元執行：powershell -NoProfile -Command "Invoke-RestMethod http://127.0.0.1:5001/status"成功時應看到 status 為 running、engine 為 WindowsOCR。
  6. 安裝成功後，「下載」資料夾中的 setup_and_run_ocr.bat 與舊版產生的 venv、logs、ocr_server.py、windows_ocr_helper.ps1 均可刪除；正式環境位於 %LOCALAPPDATA%\AI OCR Pro Editor\OCR Server，必須保留，否則自動啟動與辨識會失效。

macOS 處理步驟

開啟「終端機」、進入專案資料夾並執行；保持終端機視窗開啟：

cd /path/to/OCR
chmod +x setup_and_run_ocr.sh
./setup_and_run_ocr.sh
curl http://127.0.0.1:5001/status

成功時會看到 engine 為 AppleVision。若 5001 已被占用，先關閉舊的 OCR 伺服器視窗再重試。

iOS / Android App 處理步驟

App 顯示原生 OCR 不可用時，先確認安裝的是最新 Capacitor 打包版本而不是手機瀏覽器捷徑；重新執行 Web build 與 cap sync 後再以 Xcode/Android Studio 建置安裝。iOS 必須註冊 NativeOcrPlugin 並使用 Vision；Android 必須包含 ML Kit 中文文字辨識依賴。重新安裝後仍不可用時，先切換 Gemini/Tesseract，並將裝置型號、OS 版本與錯誤畫面提供給維護者。

安全提醒：只從本專案官方 GitHub 下載 BAT/ZIP。localhost 服務僅監聽 127.0.0.1，不需開放 Windows 防火牆公用網路；請勿為了解決連線而停用防毒或整體防火牆。

3. 編輯已辨識文字

  • 📄點擊畫布上的文字框，或點擊左側 📄 互動圖層 中的段落。被選取的文字會同步出現在右側 編輯內容 欄位。
  • ✎可直接在畫布內雙擊文字框編輯。進入編輯時，控制點會暫時隱藏，避免遮住文字內容；離開編輯後控制點會恢復。
  • ↶上方工具列的 復原 與 重做 會記錄文字、樣式、刪除與背景遮罩操作。
  • 🧩自動 OCR 遮罩只會處理像文字筆畫的像素；若系統判斷區塊過大或像背景標籤，會放棄自動遮罩以避免產生深色區塊。

4. 常用操作：區域 OCR、框選刪除與插入文字

  • ▣按 區域 OCR 後會進入 框選繪圖模式。在未辨識文字上拖曳框選，系統會只辨識該區域；若沒有辨識結果，會建立可手動輸入的文字框。
  • ⌫按 框選刪除 後會進入 框選刪除模式。框住要移除的原圖內容後，系統會以周邊背景補洞，並刪除框內文字圖層。
  • ✚從 編輯 選單點 插入文字 後，文字框不會立刻出現；請在畫布需要新增文字的位置點一下，文字框才會放在該位置。
  • ×選取文字框後按 移除文字，會移除目前文字圖層及其關聯遮罩。

5. 文字格式與本機字型

  • B右側 粗體、斜體、色彩預設 ■ □ ■ ■ ■ 🌈 可即時套用到選取文字。
  • Aa在 預設字型設定 可分別選擇 英文/數字字型 與 中文/系統字型。按 載入本機字型 後，支援的瀏覽器會列出裝置可存取字型。
  • ☑目前 圖片載入/OCR 後強制套用 預設關閉。需要統一所有 OCR 文字外觀時再手動啟用，或按 套用字型至全檔 一次套用。

6. AI 本地協同操作

  • ⓘ選取文字後按 智能校對，用於修正 OCR 錯字、空格與語句不順。旁邊的 ⓘ 可查看該功能用途。
  • ⓘ按 擷取實體 可從選取文字中整理人名、組織、指標、日期與核心概念；結果會附加到目前文字圖層。

7. 匯出與保存

  • ⬇從 檔案 選單點選 儲存圖片，會輸出已套用文字、遮罩與背景修補後的完整圖片。
  • CSV從 檔案 選單可匯出 CSV 或可搜尋 PDF；CSV 保存文字圖層內容，PDF 會把文字層加到圖片上方以利搜尋。

8. iOS / Android 使用注意

若使用真正手機端原生 OCR，請以 Capacitor 或原生 App 打包。手機瀏覽器通常不能直接啟動 localhost 原生 OCR 伺服器；打包 App 後可由 iOS Vision 或 Android ML Kit / Text Recognition 在裝置上完成辨識，不需同網段伺服器。
